# docker-rasenmaeher-integration 2.8.0

**Release date:** 2026-04-12

## TL;DR

- Added Matrix integration
  - Updated integrations to point to relevant documentation
  - Update MediaMTX integration to support SRT passwords
    - See https://github.com/pvarki/python-mediamtx-rmmtxauthz/pull/48
  - fixes https://github.com/pvarki/docker-rasenmaeher-integration/issues/241
  - Update TAKServer to 5.7

No breaking changes.

---

## Release Goals

- [Matrix](https://matrix.org/foundation/about/) is a security-oriented, powerful open-source comms suite that provides similar capabilities to eg. Signal, but in a decentralized way, which suits Deploy App very well.
- Integrations now link to the correct documentation (if it exists)
- Better stream security
- Bug fixes and new features to TAK 5.7

---

## [X] New mandatory ENV variables

  - `SYNAPSE_DATABASE_PASSWORD`
  - `SYNAPSE_MACAROON_SECRET_KEY`
  - `SYNAPSE_REGISTRATION_SECRET`
  - `RMMTX_SRT_PUB_PASSWORD`
  - `RMMTX_SRT_READ_PASSWORD`

---

## Other Notes
- Tested to be working on Element (linux) and Element Classic (android) in production and local deployments.
- The Matrix integration registers a bot user that performs user CRUD according to rmapi requests. It also auto-creates a space & rooms so that newly created & joining users automatically join there, in order to bootstrap communications for users.
- This PR adds a few new submodules:
  - `python-matrix-rmapi`: https://github.com/pvarki/python-matrix-rmapi/tree/main
  - `docker-synapse-server`: https://github.com/pvarki/docker-synapse-server/tree/main
  - `golang-keycloak-client-init-helper-cli`: https://github.com/pvarki/golang-keycloak-client-init-helper-cli
