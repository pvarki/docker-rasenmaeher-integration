# docker-rasenmaeher-integration 2.2.0

**Release date:** 2026-01-10

## TL;DR

**Adds Deploy App UI v2 & ATAK Mission Package Auto-import with the following features:**
- Retrofitted, much sleeker UI
- Style update
- Drawer/modal "welcome" guide to homepage so you know what you should do next
- The UI is now Modular which means that you can supply UI contents from product integration APIs, using Module Federation

**Adds product integration api spec to support the Modular UI:**
- You can now write UI content to your product integration API
- There's homepage and admintools content that you can write: admintools content will only show to Deploy App adminusers at the Admin Tools page:

**Adds basic new Product UIs for present products**:
- TAK modular UI, including the homepage card that contains the old startup guides & TAK Server management access at Admin Tools
- MediaMTX videosharing modular UI
- Battlelog modular UI to admintools

**Adds ATAK mission package automatic import**
- You will have a lot easier time onboarding with ATAK, just punch "Auto import package" and allow the computers to do the work for you
- As a small caveat, to integrate this functionality, we needed to allow download from a non-mTLS port, as ATAK does not seem to want to connect with  mTLS to download the package.
- For this purpose, we will use short-lived non-guessable URLs.
- Hopefully in the future ATAK will be able to use mTLS for this as well, and we can drop this functionality.

## Release Goals

- **Modular UI** is 1 of 2 requirements to actually make Deploy App fully modular, so that you as Deploy App operator can decide which products you want to deploy to your usergroups. We're now only need the modular Infra to reach this.
- **Rewritten UI** enhances user friendliness well, according to the user acceptance tests we've performed.
- New Modular product UI:s for Battlelog and MediaMTX will practically **release new products** to be used via Deploy App:
- - **Battlelog** for battlespace info management. The application is in MVP phase, so the feature/usability set is limited, and hence access to it is revealed to Deploy App instance admins only.
- - **MediaMTX** for sharing videostreams: A killer app that makes you able to create videostreams in eg. TAK using OpenTAKIcu or UASTool directly from your drone, or any streaming app that conforms to industry standards. In this MVP phase users can view the streams from Deploy App's MTX product page (list streams). Just open a stream and your usergroup sees it.
- **ATAK Mission package auto-import**:  Easier onboarding, happier users. Makes ATAK quickstart really easy, just tap one-two times and go.

## Technical Notes about the ATAK Mission package auto-import

- The short-lived non-mTLS link (served by default in port 4627) is encrypted with an instance specific key generated on container startup. The encrypted contents include the link creation time and details of the user to fetch the mission package for. The mission package contains credentials to connect to TAK and it is therefore very important that the package does not leak and that only the intended user can obtain it. To this end, the link is generated fresh when the user starts the auto-import process and the link will only be valid for 5 minutes after that time (the creation time is stored in the encrypted contents). The generated link contains quite a bit of entropy and is very long, making brute-forcing practically impossible.
- If and when TAK uses a mechanism that allows fetching this mission package with mTLS, we can retire the ephemeral link generation and close the port 4627 allocated for this purpose.
